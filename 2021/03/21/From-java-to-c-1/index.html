【C++】从java到入门c++（一）

Posted on 2021-03-21 |

前言

java和c++是开发最常用的两种语言之二，从语言学习的先后顺序上，大部分人一般先学习语法较为简单的java，在进一步深入后常会开发native相关的代码，因此c++也成为了一门躲不过去的语言。介绍c++的书很多，本系列尽量从一个java开发者入门c++的角，通过对比式的方式梳理学习c++的过程中遇到的一些关键点，供大家参考。

代码基本形式

从语言角度看，java是一种运行在虚拟机上的解释型语言，因此编译的结果是平台不相关的java字节码；c++是一种编译型语言，因此编译的时候就要指定最终运行平台支持的指令集。我们先以一个HelloWorld进行对比。

java：文件后缀为.java，文件内容以类进行组织，主函数入口为public static void main(String[] args)

c++：文件后为.cpp .hpp .cc等，本系列主要使用.cpp .h；文件内容兼容类和普通c格式；主函数入口为不含参的int main() 或含参的int main(int argc, char** args（其中argc表示参数的数量，第0个参数为改二进制程序自身，args表示参数的内容）

编译和运行

java：直接使用javac Hello.java即可得到编译好的字节码文件Hello.class，运行可以使用java Hello。

c++：编译需要使用g++或者clang++工具链，从源码到可执行文件需要经过预处理、编译、汇编、链接等过程，这里我们先关注从源码到可执行文件的外部过程，使用的命令如g++ hello.cpp -o hello，这里-o指定的是输出的可执行文件名，默认为a.out，在linux下执行该二进制需要的命令是./hello

反汇编

java：javap -c -private -v Hello.class > Hello.dump

c++：objdump -d hello > hello.dump
