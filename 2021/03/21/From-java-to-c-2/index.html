【C++】从java到入门C++（二）

Posted on 2021-03-21 |

前言

java和c++都是面向对象的编程语言，代码逻辑的组织形式是以类进行展开，设计思想上两者互通，但是具体的语法上两种语言有些差异，下面具体进行介绍。

class的定义

一个简单的c++的类定义

从上面的对比可以看出，两者在类的定义上没有本质区别，都包含成员变量、构造函数、方法这些基础的元素，下面对比一下细节上的区别：

类名：java要求public类的类名要求和所在java文件名保持一致；c++的类没有访问权限控制符，对外可见性是通过头文件进行控制的；

访问权限控制符：java的访问权限控制符是跟在每个成员变量和方法前面的，不加默认是default权限，包内的其他类可访问；c++的访问权限控制符的影响区域是一组，覆盖该控制符后面的一串变量和方法，而不加默认是private权限，只有当前类可访问；

构造方法：两种语言的构造方法都大同小异，c++有两个地方需要注意：1. 成员变量可以通过 : val(x)这种方式进行初始化； 2. 方法参数可以在定义时可以从右到左按需设置默认值，如本例中就用一个构造方法额外实现了一个不含参的构造方法；

关于析构

java和c++不同的一点在于内存管理。在java中开发者只负责创建对象和使用，哪些需要释放、如何释放以及何时都托管给虚拟机进行垃圾回收处理，缺点是GC的停顿时间会导致性能上的问题；在c++中，开发者必须主动释放自己从堆中分配的内存，否则会造成内存泄露，析构方法就是在一个类实例在被释放的时候自动调用以释放堆内存的特殊方法。

析构函数需要处理的对象主要是在当前实例通过new关键字从堆中分配的内存，比如在该Base类中构造函数中从堆中分配了一个arr指针指向一个int变量，那么在析构方法中就需要主动delete进行过释放。

注意new和delete配套使用，new []和delete []配套使用。

类的继承

java中类是单继承的，即一个类只能extends一个类，可以implements多个interface，继承之后的访问权限为：可以访问父类protect public修饰的成员变量和方法，不可以访问private；

c++中类是多继承的，可以集成多个父类，但区分public集成和私有继承（默认私有继承），如果为public继承，则可以访问父类protect和public修饰的成员变量和方法，并且继承的这些对外访问权限保持，而私有继承则将所有父类的成员变量和方法变成private的，该类对外只暴露自己的相关成员变量和方法。
