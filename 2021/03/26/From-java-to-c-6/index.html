【C++】从java到入门C++（六）- 内存泄露和野指针

Posted on 2021-03-26 |

前言

刚接触c++的时候，经常能在稳定性相关的问题定位中听到内存泄露和野指针两个关键词，对于使用java这种内存安全语言的开发者来说，这两个词可能都很陌生，但是又是c++开发者不得不面对并且要认真处理的两个问题。

内存泄露

先看这样一段简单的代码，main函数中定义了两个变量，一个是直接定义并赋值的int型变量val，另一个是通过new关键字构造出来的int型指针，虽然两者值均为3，但两者本质不同。

第一个变量val占有的是在当前栈上分配的内存，在main函数结束执行之后会被自动释放；

第二个变量newValPtr指向的是通过new关键字在堆上分配的内存，如果在main函数结束的时候不主动delete，那么对应内存的引用一直会保留，但是实际调用点已经完成了使用并且不会被再次引用，这种情况就是内存泄露。

修正：

修正版本在main函数结束前主动调用delete关键字对newValPtr指向的内存进行了释放，就不会再造成内存泄露，因此c++开发者一定要注意new和delete的配套使用，避免堆上内存的泄露。

原因在于foo中的变量val是在栈上分配的内存，当foo函数结束执行后这块内存便会被释放，而返回给main函数的指向变量val的指针此时是没有意义的，这就是野指针的一种典型场景。

后话

在java中，变量的定义和对应内存的销毁都是托管给虚拟机的，基本不会出现内存泄露问题；而且java中没有指针的概念并且对象都是在堆内，因此野指针问题也不存在。从java转向c++的开发者很容易被内存管理相关的问题困扰，但是如果用好这些关键字和语法，内存管理也可以是一种艺术。
