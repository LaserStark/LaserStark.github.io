前言

lambda表达式是一颗语法糖，帮助开发者以一种更简洁的方式进行具体实现，下面对比一下java和c++中的lambda表达式异同。

java的lambda表达式

java语言从java8开始支持lambda表达式，比如举一个常见的使用场景为例：

在这个例子中，接口MyInterface只有一个方法test，runTest方法的参数只有一个MyInterface。在runTest1中，传递的参数为MyInterface接口的一个实现MyInterfaceImpl的实例；在runTest2中，传递的参数为一个lambda表达式，这个表达式的格式为：
(Arg1 arg1, Arg2 arg2, ...) -> { expr; }
实际上替代的是实现了MyInterface接口的MyInterfaceImpl一个类。

该语法糖在不同平台上实现不同，HotSpot中是利用invoke-dynamic指令实现的；ART虚拟机中则是将其转换为一个匿名内部类。

首先看一下lambda表达式的基本语法：

  1. capture为lambda表达式需要捕获的当前函数体内的变量进行使用，[&]表示使用引用的方式捕获所有变量，[=]表示使用传值的方式进行捕获；
  2. params为传递给lambda表达式的参数，与正常的函数参数一致；
  3. opt为函数选项，比如noexcept；
  4. body为lambda表达式的函数体；
  5. ret为返回值类型，如果返回值明确则可以由编译器自行推断；
  6. 整个lambda表达式可以赋值给一个function变量，也可以auto类型由编译器进行推断；
  7. 调用时则可以使用lambda表达式变量作为函数名进行调用使用；
